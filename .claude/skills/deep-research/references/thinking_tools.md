# 사고 도구 상세 가이드

## 1. 제1원칙 사고 (First Principles)

**언제 사용하는가**: 막히거나 기존 가정이 잘못된 것 같을 때

**방법**:
1. 현재 가정들을 모두 나열
2. "이것이 반드시 참이어야 하는가?" 질문
3. 근본 원리만 남기고 재구성

**예시**:
- "API가 느리다" → "왜 느린가?" → "네트워크? 서버? 알고리즘?"
- "이 방법이 표준이다" → "왜 표준이 되었나?" → "다른 대안은?"

**적용 체크리스트**:
- [ ] 현재 모든 가정을 명시적으로 나열했는가?
- [ ] 각 가정이 정말 필수적인지 검증했는가?
- [ ] 가정 없이 처음부터 문제를 재정의했는가?

## 2. 오컴의 면도날 (Occam's Razor)

**언제 사용하는가**: 정보가 너무 많거나 복잡할 때

**방법**:
1. 모든 가능한 설명 나열
2. 불필요한 가정을 제거
3. 가장 단순한 설명 선택

**예시**:
- 버그의 원인 → 복잡한 race condition보다 단순한 typo 먼저 확인
- 성능 저하 → 복잡한 알고리즘 문제보다 단순한 캐시 미스 먼저 확인

**적용 규칙**:
```
설명1: 3개 가정 필요 → 우선순위 낮음
설명2: 1개 가정 필요 → 우선순위 높음
```

**주의**: 단순함을 추구하되, 중요한 복잡성은 무시하지 않기

## 3. 반증 가능성 (Falsifiability)

**언제 사용하는가**: 가설에 확신이 생길 때 (위험 신호!)

**방법**:
1. "이 가설이 틀렸다면 어떤 증거가 있을까?"
2. 그 증거를 적극적으로 검색
3. 발견 시 가설 수정, 미발견 시 확신도 상승

**예시**:
- "Transformer가 최고" → "Transformer가 실패한 사례 검색"
- "이 라이브러리가 안전" → "보안 취약점 검색"

**검색 쿼리 패턴**:
```
"{주장} criticism"
"{주장} wrong"
"{주장} controversy"
"{주장} limitations"
"{주장} fails when"
"{주장} alternative"
```

**확신도와 반증 탐색의 관계**:
- 확신도 50% → 지지 증거 탐색
- 확신도 70% → 반증 증거 탐색 시작
- 확신도 90% → 반증 증거 **집중** 탐색

## 4. 과학적 방법론 (Scientific Method)

**언제 사용하는가**: 새로운 방향이 필요할 때

**방법**:
1. **관찰**: 현재 알려진 사실 정리
2. **가설**: 설명 가능한 가설 제안
3. **실험**: 검색으로 가설 테스트
4. **분석**: 결과 해석
5. **수정**: 가설 업데이트

**적용 예시**:

```markdown
## 관찰
- Attention mechanism은 O(n²) 복잡도
- 긴 시퀀스에서 메모리 문제 발생

## 가설
- Linear attention이 O(n) 복잡도로 대안이 될 수 있다

## 실험 (검색)
- "linear attention transformer"
- "efficient attention mechanisms"

## 분석
- Performer, Linformer 등 다양한 시도 발견
- 정확도-효율성 트레이드오프 존재

## 수정된 가설
- Linear attention은 특정 태스크에서 효율적이나,
  원본 attention의 표현력에는 미치지 못함
```

## 5. 베이지안 추론 (Bayesian Reasoning)

**언제 사용하는가**: 증거에 따라 확신도를 업데이트할 때

**방법**:
1. 사전 확률 (Prior) 설정
2. 새 증거 발견
3. 사후 확률 (Posterior) 계산

**확신도 업데이트 규칙**:

```
신뢰도 높은 소스 지지 → +15-20%
신뢰도 중간 소스 지지 → +5-10%
신뢰도 낮은 소스 지지 → +1-3%

신뢰도 높은 소스 반박 → -20-30%
신뢰도 중간 소스 반박 → -10-15%
모순 발견 → 즉시 재검토
```

## 6. 변증법적 사고 (Dialectical Thinking)

**언제 사용하는가**: 상충하는 관점이 있을 때

**방법**:
1. **정 (Thesis)**: 현재 주장
2. **반 (Antithesis)**: 반대 주장
3. **합 (Synthesis)**: 두 관점을 통합한 새 이해

**예시**:
```
정: "Deep learning이 모든 문제를 해결한다"
반: "Deep learning은 해석 불가능하다"
합: "Deep learning은 강력하나, 해석 가능성이 필요한
     도메인에서는 하이브리드 접근이 필요하다"
```

## 도구 선택 가이드

| 상황 | 권장 도구 |
|------|-----------|
| 막힘, 진전 없음 | 제1원칙 |
| 정보 과다, 혼란 | 오컴의 면도날 |
| 높은 확신 (위험!) | 반증 가능성 |
| 새 방향 필요 | 과학적 방법론 |
| 증거 기반 업데이트 | 베이지안 추론 |
| 상충 관점 존재 | 변증법적 사고 |
