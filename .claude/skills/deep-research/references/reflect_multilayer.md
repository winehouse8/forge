# REFLECT: Multi-Layered Multi-Perspective Thinking

**목표:** Extended Thinking을 활용하여 여러 관점에서 동시에 깊이 분석하고, 변증법적 토론을 통해 통합된 이해에 도달

**기반 연구 (2025-2026):**
- [Matrix of Thought](https://arxiv.org/html/2509.03918) - 다중 분기 추론
- [Reasoning through Perspective Transition](https://arxiv.org/html/2501.09265v1) - 3가지 관점
- [Roundtable Reasoning](https://arxiv.org/abs/2509.14834) - 변증법적 통합
- [Metacognition in LLMs](https://arxiv.org/html/2509.21545v1) - 메타인지 프레임워크
- [Hegelion - Dialectical Reasoning](https://github.com/Hmbown/Hegelion) - Thesis-Antithesis-Synthesis

**Quick Scan:**

```markdown
- 현재 iteration 번호: N
- 지금까지 알게 된 핵심 사실: [상위 5개]
- 아직 모르는 것: [미해결 질문 3개]
- Active Hypotheses 상태: [Top 5 가설 + 확신도]
- 최근 발견의 신뢰도: [✓✓/✓/~/? 분포]
```

**목적:** 현재 위치 빠르게 파악

## Layer 2: Multi-Perspective Reasoning (다중 관점 병렬 사고)

**Extended Thinking으로 3가지 관점을 동시에 탐색:**

### Perspective 1: Direct Perspective (내부 이해)

*"내가 현재 이해한 바로는..."*

**질문:**
- 현재 가설들이 논리적으로 일관된가?
- 증거들이 자연스럽게 연결되는가?
- 내부적으로 모순은 없는가?
- 직관적으로 "뭔가 이상한" 부분은?

**Output:** [현재 내부 이해 요약]

**예시 (양자 컴퓨팅):**
```
내부 이해:
- 양자 큐비트는 중첩 상태로 존재 → 논리적 일관
- 측정 시 상태 붕괴 → 자연스럽게 연결
- 모순: 없음
- 이상한 점: 오류율이 예상보다 높음 (추가 조사 필요)
```

---

### Perspective 2: Role Perspective (전문가 역할)

*"만약 내가 [도메인 전문가]라면..."*

**전문가 역할 선택:**

현재 주제에 적합한 3-5개 전문가 역할 선택

**예시:**
- 양자 컴퓨팅 → [물리학자, 컴퓨터 과학자, 공학자, 회의론자, 투자자]
- AI 윤리 → [철학자, 법률가, 기술자, 사회학자, 피해자]
- 경제 정책 → [경제학자, 정치인, 기업인, 노동자, 소비자]

**각 역할별 질문:**

| 역할 | 핵심 질문 | 예시 (양자 컴퓨팅) |
|------|----------|-------------------|
| **물리학자** | 이론적 기반이 탄탄한가? | 양자역학 원리에 부합하는가? |
| **공학자** | 실제 구현이 가능한가? | 상온에서 작동 가능한가? |
| **회의론자** | 어디서 틀릴 수 있는가? | 오류율 문제 해결 불가능? |
| **투자자** | 비용 대비 가치는? | ROI가 정당화되는가? |
| **사용자** | 실제 도움이 되는가? | 일상에 어떤 영향? |

**Output:** [각 전문가 관점에서의 평가]

---

### Perspective 3: Third-Person Perspective (외부 관점)

*"제3자가 내 연구를 보면..."*

**비판적 질문:**
- 편향은 없는가?
  - 확증 편향 (지지 증거만 찾음)
  - 생존자 편향 (성공 사례만 봄)
  - 권위 편향 (유명 출처만 신뢰)
- 놓친 대안 가설은 없는가?
- 검색 전략의 맹점은?
- 다른 연구자라면 어떤 질문을 할까?

**Output:** [외부 관점 비판]

**예시:**
```
외부 관점 비판:
- 확증 편향 의심: 양자 컴퓨팅 성공 사례만 찾음
- 놓친 대안: 고전 컴퓨팅 최적화가 더 효율적일 수도
- 검색 맹점: "양자 컴퓨팅 실패 사례" 미검색
- 다른 연구자 질문: "비용이 정당화되는가?"
```

## Layer 3: Dialectical Synthesis (변증법적 통합)

**Roundtable Discussion - 3가지 관점의 토론:**

### Phase A: Thesis (정)

"Perspective 1 (Direct)의 결론을 채택하면..."

**주장:**
- [Direct Perspective의 핵심 주장]

**근거:**
- [지지 증거 3개]

**예시:**
```
주장: 양자 컴퓨팅은 특정 문제에서 고전 컴퓨팅을 압도한다

근거:
1. Shor's algorithm: 소인수분해 지수적 가속
2. Grover's algorithm: 검색 속도 제곱근 가속
3. 양자 시뮬레이션: 화학/물리 시뮬레이션 정확도 향상
```

---

### Phase B: Antithesis (반)

"하지만 Perspective 2 (Role)과 3 (Third-Person)에서 보면..."

**반론:**
- Perspective 2의 비판: [전문가들의 우려]
- Perspective 3의 비판: [외부 관점의 맹점 지적]

**반증 증거:**
- [모순되는 증거 2개]

**예시:**
```
반론:
- 공학자: 오류율 1%로는 실용 불가능
- 투자자: 초기 비용이 너무 높음 (수억 달러)
- 외부 관점: 성공 사례는 극소수, 대부분 실패

반증 증거:
1. IBM 양자 컴퓨터: 실용 문제 해결 실패 사례 많음
2. 고전 알고리즘 개선으로 양자 우위 일부 상쇄
```

---

### Phase C: Synthesis (합)

"두 관점을 통합하면..."

**통합된 이해:**
- Thesis의 장점: [수용할 부분]
- Antithesis의 통찰: [수정할 부분]
- 새로운 중간 지점: [통합 결론]

**남은 불확실성:**
- [여전히 모르는 것 명시]

**Output:** [통합된 이해 + 불확실성 리스트]

**예시:**
```
통합된 이해:
- Thesis 장점: 이론적으로 특정 문제에서 확실한 우위 있음
- Antithesis 통찰: 현재 기술로는 실용화 어려움
- 통합 결론: 양자 컴퓨팅은 특정 niche 분야에서 유망하나,
              범용 컴퓨팅 대체는 10-20년 이상 소요 예상

남은 불확실성:
1. 오류 수정 기술 발전 속도
2. 실용화 시점 (5년? 10년? 20년?)
3. 고전 알고리즘 개선 vs 양자 발전 경쟁 결과
```

## Layer 4: Meta-Cognitive Monitoring (메타인지)

### Self-Assessment Checklist

**사고 품질 체크:**

```markdown
□ **논리적 일관성**: 내 추론 과정에 모순은 없는가?
  - A를 주장하면서 동시에 not-A 주장?
  - 전제와 결론이 연결되는가?

□ **증거 품질**: 신뢰할 만한 출처인가?
  - ✓✓ (다수 신뢰 소스) 비율은?
  - Peer-reviewed vs Blog 비율은?

□ **편향 체크**: 확증 편향에 빠지지 않았는가?
  - 반대 증거도 적극적으로 찾았는가?
  - 불편한 데이터를 무시하지 않았는가?

□ **대안 고려**: 다른 설명 가능성을 충분히 탐색했는가?
  - 최소 3개 이상 대안 가설 검토?
  - 왜 현재 가설이 최선인가 설명 가능?

□ **맹점 인식**: 내가 놓치고 있는 것은 무엇인가?
  - 검색하지 않은 키워드는?
  - 고려하지 않은 관점은?
```

---

### Zoom Out / Zoom In Check

#### 🔭 Zoom Out (거시적)

**전체 맥락:**
- 전체 연구 질문에서 현재 위치는?
  - 예: "5개 서브질문 중 2개 답변 완료 (40%)"
- 큰 그림에서 놓친 영역은?
  - 예: "사회적 영향은 조사했지만 환경적 영향 미조사"
- 다른 분야와의 연결은? (학제간 관점)
  - 예: "양자 컴퓨팅 + 암호학 연결 확인, but + 의료 분야 미확인"

**예시:**
```
거시적 관점:
- 현재 위치: 기술적 타당성 확인 (3/5 완료)
- 놓친 영역: 규제, 윤리적 측면 미조사
- 학제간: 물리+CS 연결 OK, 경제학 연결 미흡
```

---

#### 🔬 Zoom In (미시적)

**세부 검증:**
- 이 증거의 세부 논리는 탄탄한가?
  - 전제 → 결론 연결이 논리적인가?
  - 숨겨진 가정은 없는가?
- 출처의 신뢰도는?
  - Peer-reviewed? Preprint? Blog?
  - 저자의 전문성은?
  - 이해 상충은 없는가?

**예시:**
```
미시적 검증:
- 증거 "Shor's algorithm 검증":
  - 전제: 양자 컴퓨터가 충분한 큐비트 보유
  - 결론: 소인수분해 빠름
  - 숨겨진 가정: 오류율이 충분히 낮음 ← 확인 필요!

- 출처 신뢰도:
  - Nature 논문 (peer-reviewed) ← 높음
  - 기업 블로그 (홍보성) ← 낮음
```

---

#### ↔️ Zoom 왕복

**일관성 체크:**
- 세부 증거가 큰 그림과 일치하는가?
  - 예: "세부적으로는 성공했지만, 전체 목표 달성은 미흡"
- 거시적 패턴이 미시적 증거로 뒷받침되는가?
  - 예: "양자 우위 주장하지만, 개별 실험은 제한적"

**Output:** [메타인지 체크 결과 + 개선 포인트]

## Layer 5: Thinking Tools Selection (사고 도구 선택)

### 현재 상황 진단

```markdown
- 막힘 여부: [Yes/No]
  - 5회 이상 진전 없음? → Yes
- 정보 과부하: [Yes/No]
  - 100개 이상 소스, 정리 안됨? → Yes
- 확신도 >80%: [Yes/No]
  - 주요 가설 확신도 체크
- 새 방향 필요: [Yes/No]
  - 현재 접근법 막다른 길? → Yes
```

---

### 적용할 사고 도구 선택

| 상황 | 도구 | 적용 |
|------|------|------|
| 막힘 | **제1원칙** | 가정 분해 → 원리 재구성 |
| 정보 과다 | **오컴의 면도날** | 단순 설명 우선 |
| 확신도 >80% | **반증 가능성** | 반박 증거 적극 탐색 |
| 새 방향 | **SCAMPER** | Substitute, Combine, Adapt, Modify, Put to use, Eliminate, Reverse |
| 모순 발견 | **베이지안 추론** | 확률 업데이트 |
| 혁신 필요 | **TRIZ** | 모순 해결, 40가지 발명 원리 |

**상세:** [thinking_tools.md](./thinking_tools.md)

**Output:** [선택된 도구 + 적용 계획]

## Layer 6: Matrix of Thought (다중 경로 탐색)

### 목표

단일 추론 경로 대신 **3개 병렬 경로** 탐색하여 교차 검증

```
      [현재 상황]
           ↓
    ┌──────┼──────┐
    ↓      ↓      ↓
  경로A  경로B  경로C
(정통)  (대안) (반대)
    ↓      ↓      ↓
  결론A  결론B  결론C
    └──────┼──────┘
           ↓
     [교차 검증]
```

---

### 경로 A (정통 접근)

**설명:** 가장 명백한 추론 경로

**방법:**
- 기존 가설 기반 전개
- 일반적으로 받아들여지는 논리
- 주류 증거 활용

**결론:** [A]

**예시:**
```
정통 접근:
- 가설: 양자 컴퓨팅은 고전 컴퓨팅보다 우월
- 논리: Shor's/Grover's algorithm 증명
- 증거: Nature 논문, IBM/Google 발표
- 결론: 특정 문제에서 양자 우위 확실
```

---

### 경로 B (대안 접근)

**설명:** 다른 각도에서 접근

**방법:**
- 역발상 (Reverse thinking)
- Lateral thinking (SCAMPER)
- 비주류 증거 활용

**결론:** [B]

**예시:**
```
대안 접근:
- 역발상: 고전 컴퓨팅 최적화가 더 효율적?
- Lateral: 뉴로모픽 컴퓨팅 대안은?
- 비주류: 중국 연구팀의 다른 결과
- 결론: 양자 vs 고전은 문제 의존적, 범용 우위 없음
```

---

### 경로 C (반대 접근)

**설명:** 현재 가설의 반대편에서 시작 (Devil's Advocate)

**방법:**
- 현재 가설의 정반대 주장
- 반증 증거 적극 활용
- "왜 틀렸는가" 관점

**결론:** [C]

**예시:**
```
반대 접근:
- 반대 주장: 양자 컴퓨팅은 과대평가됨
- 반증: 실용 문제 해결 실패 사례 다수
- Devil's Advocate: 오류율 해결 불가능?
- 결론: 현재 기술로는 양자 우위 미미, 10년 내 실용화 어려움
```

---

### 교차 검증

**일치 부분 (공통점):**
- 경로 A, B, C가 모두 동의하는 부분
- → **신뢰도 높음**

**충돌 부분 (모순점):**
- 경로들이 상반된 결론 내린 부분
- → **추가 검증 필요**

**최종 선택:**
- 가장 신뢰할 만한 경로 선택 + 이유

**예시:**
```
교차 검증:
- 공통점: 모든 경로가 "이론적 가능성" 인정
- 충돌점: 실용화 시점 (A: 5년, B: 10년, C: 20년+)
- 최종 선택: 경로 B 채택
  - 이유: 성공/실패 사례 모두 고려, 가장 균형잡힌 관점
```

**Output:** [3개 경로 탐색 + 최종 선택]
