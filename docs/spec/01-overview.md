## 제품 정의

### 이름
**Pathfinder Deep Research Skill**

### 목적
사용자가 명시적으로 중단할 때까지 자동으로 심층 연구를 수행하는 **자율 AI 연구 에이전트**.

LLM의 주관적 "완료" 판단을 무시하고 **객관적 조건만**으로 종료를 제어하는 **Ralph Loop 패턴**을 구현.

### 핵심 문제
기존 AI 연구 도구의 한계:
- ❌ LLM이 주관적으로 "충분하다"고 판단하면 조기 종료
- ❌ 95% 확신도에서도 반증 증거 탐색 중단
- ❌ 중단 후 재개 불가
- ❌ 정보 출처 불명확, 신뢰도 평가 없음

### 솔루션
- ✅ Ralph Loop 패턴으로 무한 반복
- ✅ 높은 확신도에서도 계속 반증 탐색
- ✅ 상태 지속성으로 중단 후 재개 가능
- ✅ 4계층 검증 시스템, 소스별 신뢰도 평가

## 주요 사용 사례

### 1. 복잡한 기술 주제 조사

**시나리오:**
```
질문: "양자 컴퓨팅의 최신 동향은?"

연구 프로세스:
1. 일반 웹 검색 (최신 뉴스, 블로그)
2. 학술 검색 (arXiv 논문)
3. 반증 검색 ("양자 컴퓨팅 한계", "실패 사례")
4. 교차 검증 (정보 일치 여부)
5. 지식 그래프 구축 (개념 간 관계)
6. 가설 수립 및 검증
...
(사용자가 중단할 때까지 반복)
```

**결과:**
- 다각도 분석 (긍정/부정/중립)
- 검증된 사실 (✓✓ 태그)
- 불확실한 주장 (? 태그)
- 모순 탐지 (⚠ 태그)

---

### 2. 학술적 문헌 리뷰

**시나리오:**
```
질문: "Transformer 아키텍처의 발전 과정"

연구 프로세스:
1. arXiv 검색 ("Transformer architecture")
2. PDF 다운로드 및 분석
3. 인용 논문 추적
4. 연도별 발전 과정 정리
5. 핵심 기여 요약
...
```

**결과:**
- 연대기 순 논문 목록
- 각 논문의 핵심 기여
- 인용 관계 그래프
- 논문 다운로드 (`papers/` 디렉토리)

---

### 3. 다각도 사실 검증

**시나리오:**
```
주장: "GPT-4는 2023년 3월 출시되었다"

검증 프로세스:
1. 지지 증거 검색
   - OpenAI 공식 블로그
   - 기술 뉴스 사이트
2. 반증 증거 검색
   - "GPT-4 release date wrong"
   - "GPT-4 출시일 논란"
3. 교차 검증
   - 3개 이상 독립 소스 확인
4. 신뢰도 평가
   - openai.com: 0.98 (고신뢰)
   - techcrunch.com: 0.85
```

**결과:**
```
✓✓ VERIFIED: GPT-4는 2023년 3월 14일 출시
   소스: openai.com, techcrunch.com, theverge.com
   신뢰도: 0.95
```

---

### 4. 지식 그래프 구축

**시나리오:**
```
주제: "머신러닝 개념 체계"

연구 프로세스:
1. 핵심 개념 추출
   - 노드: Supervised Learning, Neural Networks, ...
2. 관계 정의
   - 엣지: "Supervised Learning enables Classification"
3. 신뢰도 부여
   - 각 관계의 신뢰도 점수
4. 모순 탐지
   - 충돌하는 정보 발견 및 표시
```

**결과:**
```json
{
  "nodes": [
    {"id": "ml_001", "label": "Machine Learning", "confidence": 0.98},
    {"id": "sl_001", "label": "Supervised Learning", "confidence": 0.95}
  ],
  "edges": [
    {"from": "sl_001", "to": "ml_001", "relation": "is_subset_of", "confidence": 0.97}
  ]
}
```

## 제약사항

### 기술적 제약

| 제약 | 설명 | 영향 |
|------|------|------|
| **실행 환경** | Claude Code 필수 | 다른 환경에서 실행 불가 |
| **API 의존** | WebSearch/WebFetch 필수 | 오프라인 실행 불가 |
| **단일 세션** | `.research/` 디렉토리 공유 | 동시 연구 불가 |
| **토큰 제한** | 32K output, 10K thinking | 매우 긴 응답 제한 |

### 비용 제약

**기본 예산:** $10/세션

**예상 비용 (100 iterations):**
- Input: ~300K tokens × $0.003/1K = $0.90
- Output: ~200K tokens × $0.015/1K = $3.00
- **총: ~$3.90**

**실제 변동:**
- Extended Thinking 사용 시 +50%
- PDF 분석 시 +30%
- 복잡한 질문 시 +40%

### 시간 제약

- **Iteration당:** 2-3분
- **100 iterations:** 3-5시간
- **적합 사용 사례:** 실시간 연구 (수 시간)
- **부적합:** 배치 처리 (며칠)
